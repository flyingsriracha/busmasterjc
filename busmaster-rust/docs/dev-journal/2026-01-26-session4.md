### Implementation Details

#### DoIP Protocol Overview
DoIP (Diagnostics over IP) is defined in ISO 13400 and provides diagnostic communication over IP networks. It's commonly used in modern vehicles for Ethernet-based diagnostics.

#### Key Components Implemented

1. **DoipHeader** - 8-byte header with:
   - Protocol version (0x02 for ISO 13400-2:2012)
   - Inverse version (for validation)
   - Payload type (16-bit)
   - Payload length (32-bit)

2. **Payload Types** - All standard types:
   - Vehicle identification (request/response)
   - Routing activation (request/response)
   - Alive check (request/response)
   - Diagnostic message (UDS tunneling)
   - Entity status
   - Power mode information
   - Negative acknowledge

3. **Response Codes**:
   - Generic NACK codes
   - Routing activation response codes
   - Diagnostic message ACK/NACK codes

4. **DoipClient** - State machine with:
   - Disconnected → Connected → RoutingActivated states
   - Helper methods for creating requests
   - Response processing with state updates
