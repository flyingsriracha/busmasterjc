- Implemented complete CAN XL support in busmaster-core and busmaster-proto
- Created `CanXlFrame` struct with:
  - Up to 2048 bytes of data (vs 64 for CAN FD)
  - Priority field (0-255, lower = higher priority)
  - Virtual CAN Network ID (VCID) for network segmentation
  - Service Data Unit Type (SDT) enum with ContentBasedAddressing, ClassicalCanTunneling, CanFdTunneling, Ethernet, IPv4, IPv6, Reserved
  - Acceptance Field (32-bit) for filtering
  - Simple Extended Content (SEC) flag
  - Word alignment requirement for data lengths > 48 bytes
- Implemented backward compatibility:
  - `to_canfd()` - Convert to CAN FD if data <= 64 bytes
  - `to_can()` - Convert to classical CAN if data <= 8 bytes
  - `from_canfd()` - Create CAN XL from CAN FD frame
  - `from_can()` - Create CAN XL from classical CAN frame
- Added CAN XL parsing/encoding in busmaster-proto:
  - `CanParser::parse_xl()` - Parse CAN XL from bytes
  - `CanEncoder::encode_xl()` - Encode CAN XL to bytes
  - `CanEncoder::encoded_size_xl()` - Calculate encoded size
  - Wire format with 24-byte header + data
